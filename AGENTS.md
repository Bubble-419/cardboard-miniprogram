# 项目说明
你是一个专业的微信小程序开发者，现在请帮我开发一款协助卡牌游戏流程控制的小程序。开发过程中需要严格遵循微信小程序开发的语法和规则。
这个小程序主要是用于协助卡牌游戏进行流程控制，卡牌游戏的目的是让多名玩家进行头脑风暴。
小程序的用户有两种角色，第一种是上帝：直接用小程序来控制游戏流程，上帝用户会进入主屏页面。第二种是玩家：玩家用户会进入副屏页面，副屏页面展示的内容受主屏的控制（因为主屏控制流程，副屏页面根据不同的流程而改变）。

# 前端规则
- 我会从figma选择对应的界面frame并告知你，请你严格按照figma的设计进行coding，并保证是可以交互的。
- 主屏和副屏之间存在通信规则，请严格保证副屏会按照通信规则跳转页面。
- 请及时将复用性高的代码块抽成组件，并更新至components文件夹。

# 功能需求
## 用户登陆和鉴权
通过一个页面让用户选择自己是作为上帝用户还是玩家用户（选择上帝用户则后续跳转至主屏页面，选择玩家用户则后续跳转至副屏页面）。
## 选择设计问题
由玩家用户在副屏页面（submitProblem）提交自己的设计问题，上帝用户可以在主屏页面（selectProblem）实时接收到已提交的设计问题。在上帝用户选择设计问题时，玩家用户也会跳转至副屏的selectProblem页面（和主屏一致只是玩家用户无法交互，即无法做出选择）。
## 选择模式
由上帝用户在主屏页面（selectMode）选择模式，此时玩家用户进入等待页面（awaitMode）。
## 选择出牌玩家
由上帝用户在主屏页面（selectPlayer）选择出牌玩家，此时玩家用户进入等待页面（awaitPlayer）。


# 项目架构
## 文件夹说明
### 一级文件夹：
- pages：用于存放小程序的页面。
- components: 用于存放公共组件。
### 二级文件夹：
- auth: 用于存放用户登陆和鉴权页面。
- main-pages: 用于存放主屏页面。
- sub-pages: 用于存放副屏页面。
### 主屏三级文件夹（针对上帝用户）：
- selectProblem：选择设计问题页面。
- selectMode：选择模式。
- selectPlayer：选择出牌玩家。
### 副屏三级文件夹（针对玩家用户）：
- selectProblem：提交设计问题页面。
- awaitMode：等待上帝用户选择设计问题页面。
- awaitPlayer：等待上帝用户选择模式页面。
